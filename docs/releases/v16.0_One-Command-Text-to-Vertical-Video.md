## 1. The Goal: One Text → One Video

The annoying part of making short-form talking-head videos isn't writing the script. It's everything after:

Feed it into a TTS tool to synthesize audio. Export the transcript into an SRT file. Hand-write a FFmpeg filter to stitch the video, the audio, and the subtitles together. Four tools, three copy-pastes, one typo and you start over.

Can it be compressed into **one command**?

Yes. Here's the pipeline:

```
Text → IndexTTS2(voiceover) → FunASR(transcribe→SRT) → FFmpeg(stitch video+audio+subs)
```

Input: a paragraph of Chinese text. Output: a 1080×1920 vertical MP4 with voiceover, subtitles, and a filled background. The whole chain runs end-to-end in about **30 seconds**.

---

## 2. The Chain, Step by Step

### Step 1 — IndexTTS2 Voiceover

```
python full_chain.py "your script text" --ref ref.wav
```

IndexTTS2 runs locally, `use_fp16=True, device="cuda:0"`. Give it a reference audio clip (`ref.wav`) and it reads your text in that voice.

Benchmark: a 12-second script synthesized in **15–16 seconds, RTF 0.70** — slightly faster than real time.

> RTF (Real-Time Factor) = synthesis time ÷ audio length. Below 1 means faster than the audio plays back.

### Step 2 — FunASR Transcription to SRT

Take the synthesized audio and feed it to FunASR for captions:

```python
from funasr import AutoModel
asr = AutoModel(model="paraformer-zh", device="cuda:0", disable_update=True)
res = asr.generate(input="tts.wav", batch_size=1)
```

The `paraformer-zh` model, Chinese speech recognition. Benchmark: **4 seconds, RTF 0.08** — practically free. Accuracy on TTS-generated audio is perfect because the source is clean, zero background noise.

Output: an SRT file for FFmpeg to burn in.

### Step 3 — FFmpeg Composition

This is where the real engineering lives. Three things to do:

1. Scale the source video to fill 1080×1920 (`scale` + `pad`, since the source might be any aspect ratio)
2. Layer the TTS audio on top
3. Burn the subtitle text onto the bottom of the frame (`drawtext`)

The filter chain:

```bash
[0:v]scale=1080:1920:force_original_aspect_ratio=decrease,
pad=1080:1920:(ow-iw)/2:(oh-ih)/2:#0F172A[bg];
[bg]drawtext=textfile=subtitle.txt:fontsize=36:fontcolor=white:
x=(w-text_w)/2:y=h-120:box=1:boxcolor=black@0.7:boxborderw=8[v];
[1:a]aformat=sample_rates=48000:channel_layouts=mono[a]
```

Background color `#0F172A` — dark navy, matches the site's dark-mode palette. Subtitles get a semi-transparent black box for legibility on any background.

If the voiceover is shorter than the video, the script auto-pads with silence to match lengths.

## 4. The Biggest Bug: FFmpeg's `:` vs. Windows' `E:/`

This was the nastiest bug on the chain. It took me half a day.

In FFmpeg's `filter_complex` syntax, **the colon `:` is a parameter delimiter**.

In a Windows path `E:/workout/pipeline/video.mp4`, **the second character is a colon**.

FFmpeg sees `E:`, treats it as a filter parameter, and the entire filter chain blows up.

I tried five approaches:

| Approach | Result |
|----------|--------|
| `file:///` URI prefix | FFmpeg doesn't recognize it |
| Pass an ASS subtitle file | Colon still parsed as parameter |
| `drawtext` inline `text=` | Chinese special characters get mangled |
| `drawtext textfile=` | Path colon still explodes |
| **✅ Convert everything to relative paths** | Works |

The fix: every path going into FFmpeg (input video, audio, output file) gets converted to a relative path. Working directory `cwd` is pinned to `E:\workout\pipeline`. No colons in paths, filter chain runs clean.

Encapsulated as a `_rp()` helper:

```python
def _rp(p):
    """Convert to relative path to avoid E:/ colon breaking FFmpeg filters."""
    if p.startswith("/e/"):
        p = "E:/" + p[3:]
    if os.path.isabs(p):
        return os.path.relpath(p, FFMPEG_CWD).replace("\\", "/")
    return p
```

Every path argument to FFmpeg goes through `_rp()` first. Colons are dead.

Secondary bug fixed along the way: `step1_tts` originally called `os.chdir()` to switch directories for IndexTTS, which polluted FFmpeg's `cwd`. Switched to passing absolute paths directly to IndexTTS — `cwd` stays clean.
