# LTX-2B I2V on 8 GB: From Broken CLIP to a 6-Second 3-Segment Relay — and Why One Node Saved the Video

> Date: 2026-07-15 · GPU: RTX 5060 Laptop 8GB · CUDA: cu132 · Framework: ComfyUI 0.24.0 · OS: Windows 11 native

## TL;DR

LTX-2B is one of the only open-source video generation models that actually fits on an 8 GB card. The model itself is 6 GB, and with the right node chain a 49-frame (2.04s) segment generates in about 18 seconds — three segments chained end-to-end gives a 6-second clip for roughly one minute of compute.

But LTX-2B's first impression is hostile. The default CLIP text encoder returns `None`. The LTX-AV text encoder demands a tokenizer that doesn't exist in this version. And without two obscure stabilizer nodes — STG and StatNorm — the video drifts so badly by frame 30 that the subject barely resembles the source image.

The headline numbers: **49 frames / 18 seconds / junction SSIM 0.96+** (with the official ComfyUI-LTXVideo extension installed).

This article documents the dead ends, the working node chain, and the single node that separates "usable" from "warping apart frame by frame."

---

## What Is LTX-2B

LTX-2B (Lightricks) is a 2.1B-parameter image-to-video DiT trained to extend a still frame into a short coherent clip. The v0.9.5 checkpoint (6 GB) runs on 8 GB VRAM with `--lowvram`, producing 768×512 video at 24 fps.

What makes it unusual for consumer hardware:

| Property | Value |
|----------|-------|
| Model size | 6 GB (safetensors) |
| Max resolution | 768×512 (native), larger via tiling |
| Frame range | 33–49 frames practical on 8 GB |
| Text encoder | T5 XXL fp8 (not CLIP) |
| Native LoRA | Not supported |
| Built-in keyframe locking | Yes (image-to-video conditioning) |

The keyframe locking is what makes LTX-2B useful for multi-segment relay: the model anchors each new segment to the last frame of the previous one, so you can chain segments without cumulative subject drift — *provided* the stabilizers are engaged.

## The ComfyUI Trap: "CLIP" Returns Nothing

Most image models in ComfyUI use a CLIP text encoder baked into the checkpoint. LTX-2B does not. If you drag a standard `CLIPTextEncode` node onto the LTX checkpoint, it returns `None` silently — no error, just empty conditioning. The sampler will run and produce noise.

The first dead end:

| Attempt | Result |
|---------|--------|
| `CLIPTextEncode` from checkpoint clip | ❌ LTX checkpoint has no CLIP weights → returns `None` |
| `LTXAVTextEncoderLoader` (type=ltxv) | ❌ Built for LTX-2.3, requires Gemma 3 tokenizer → "invalid tokenizer" |
| **`CLIPLoader(type=ltxv, clip_name=t5xxl_fp8_e4m3fn.safetensors)`** | ✅ **Works** — T5 XXL loaded, text encoded, pipeline runs |

The working path requires a standalone T5 XXL fp8 file (`t5xxl_fp8_e4m3fn.safetensors`, ~3.8 GB) in `models/text_encoders/`, and the CLIPLoader must be told `type=ltxv`. There is no universal setting — the wrong type silently falls through to a CLIP tokenizer that LTX doesn't speak.

## The Minimal Working Node Chain

This is the chain that generates a clean 49-frame segment on 8 GB:

```
CheckpointLoaderSimple(ltx-video-2b-v0.9.5.safetensors)
  → CLIPLoader(type=ltxv, clip_name=t5xxl_fp8_e4m3fn.safetensors)
    → CLIPTextEncode(positive: "subtle gentle movement, slight head turn")
    → CLIPTextEncode(negative: "occluded face, camera movement, orbit, rotation")
    → LTXVImgToVideo(image=source, strength=0.45, resolution=768×512, frames=49)
    → LTXVConditioning(frame_rate=24)
    → KSampler(model, steps=30, cfg=1.0, scheduler=simple, seed=777777)
    → VAEDecode(ltx_video_vae.safetensors)
    → SaveAnimatedPNG(fps=24, compress=4)
```

**Critical parameters:**

| Parameter | Value | Why |
|-----------|-------|-----|
| strength | 0.40–0.45 | Above 0.50: motion visible but face starts distorting |
| steps | 30 | Diminishing returns above 35; below 20 quality drops |
| cfg | 1.0 | LTX-2B uses guidance-free sampling; higher cfg breaks coherence |
| scheduler | simple | Karras and other schedulers tested worse on this checkpoint |
| seed | 777777 | Stable across runs |
| frames | 49 | 8 GB practical maximum; 33 is safer |

The VAE must be LTX's native `ltx_video_vae.safetensors` — an SDXL VAE silently produces color-shifting artifacts.

**Runtime: ~18 seconds per 49-frame segment** on RTX 5060 8GB with `--lowvram`.

## The STG / StatNorm Gap: What Breaks When They're Missing

ComfyUI's built-in LTX nodes work, but they lack two stabilizer nodes that the official extension provides. The difference is dramatic — and measurable:

| Metric | Built-in nodes (no stabilizers) | With ComfyUI-LTXVideo extension |
|--------|:-------------------------------:|:-------------------------------:|
| Intra-segment mean SSIM | **0.8321** | **0.8862** |
| Intra-segment worst jump | 0.7261 | 0.6224 |
| End-to-end (first→last frame) | 0.5583 | 0.5261 |
| **Segment junction SSIM** | **0.9639** | 0.5260 |
| Cumulative drift after 3 segments | Severe | Minimal |

Two distinct problems, each solved by a different node:

**1. STGGuiderAdvanced (Spatio-Temporal Grouping)**

STG applies attention regularization across the Transformer's temporal layers. Without it, the model gradually loses the spatial identity of the subject — the face still exists, but features drift: mouth shape softens, hairline shifts, eyes lose symmetry. The SSIM mean of 0.83 captures this: frames are "similar" in a statistical sense but no longer depict the same person.

With STG layers [0, 47] and STG scale = 2.0, the intra-segment mean climbs to 0.8862.

**2. LTXVStatNormLatent**

StatNorm applies statistical normalization to the latent space after each denoising step. Without it, the latent mean and variance slowly walk away from the source, causing the subject to "fade" into noise toward the back of the clip. A factor of 1.0 keeps the latent anchored to the source frame's distribution.

The combined effect: with both nodes engaged, a 3-segment (145-frame) relay is visually coherent. Without them, the third segment looks like a different person.

**Counter-intuitive finding:** the *junction* SSIM is actually *better* in the built-in pipeline (0.96 vs 0.53). The built-in LTXVImgToVideo references the previous segment's last frame as its source image, which locks the first frame tightly. The old pipeline's LoopingSampler didn't anchor the first frame as aggressively, so its segment boundaries popped. The modern approach wins on junctions; it loses on intra-segment stability.

## Installing the Official Extension

The official Lightricks extension (`Lightricks/ComfyUI-LTXVideo`) provides the missing nodes. As of 2026-07-15, the repository is active (3949 stars).

```bash
cd E:/WSL/ComfyUI/custom_nodes
git clone --depth 1 https://github.com/Lightricks/ComfyUI-LTXVideo.git
cd ComfyUI-LTXVideo
pip install "ninja>=1.11,<2.0" diffusers einops transformers kornia
```

Then restart ComfyUI. The new nodes appear in the node list:

| Node | Purpose |
|------|---------|
| `LTXVApplySTG` | Apply STG guidance between denoising steps |
| `LTXVStatNormLatent` | Statistical normalization of the latent |
| `STGGuiderAdvanced` | Advanced STG with configurable layer range |
| `LTXVLoopingSampler` | Looping sampler with noise mask control |
| `LTXVTiledVAEDecode` | Tiled VAE decode for higher resolutions |

The patched chain (built-in + extension):

```
...KSampler output...
  → STGGuiderAdvanced(layers=[0,47], stg_scale=2.0)
  → LTXVStatNormLatent(factor=1.0)
  → LTXVLoopingSampler(noise_mask)
  → LTXVTiledVAEDecode
  → SaveAnimatedPNG
```

Re-running with STG + StatNorm restores the intra-segment SSIM to 0.88+ while keeping the junction SSIM at 0.96+.

## Three-Segment Relay

The keyframe locking property lets you chain segments by feeding each segment's output as the next segment's input:

```
Segment A: source_image → 49f (0–2.04s)
Segment B: last_frame_of_A → 49f (2.04–4.08s)
Segment C: last_frame_of_B → 49f (4.08–6.12s)
Concatenate → 145 frames, 6.04 seconds
```

Total runtime: **~1 minute** for a 6-second clip on 8 GB.

Output files (E:/WSL/ComfyUI/output/):
```
ltx_segA_49f_00001_.png   # Animated PNG, 49 frames
ltx_segB_49f_00001_.png
ltx_segC_49f_00001_.png
ltx_v7_3segs_6s.mp4       # Concatenated MP4, 145 frames, 6.04s
```

Note: ComfyUI's `SaveAnimatedPNG` outputs individual animated PNGs, not MP4s. Concatenation requires a post-process ffmpeg step or the ComfyUI VideoHelperSuite nodes.

## Lessons

| Lesson | Detail |
|--------|--------|
| **LTX does not use CLIP** | Standard `CLIPTextEncode` returns `None`. Use `CLIPLoader(type=ltxv)` with a standalone T5 XXL fp8 file. |
| **`--lowvram` is mandatory** | The 6 GB model + T5 XXL (3.8 GB) + dequantization peak exceed 8 GB without it. |
| **STG + StatNorm are not optional** | Without them, frame 30 looks different from frame 0. The official extension provides both. |
| **CFG must be 1.0** | LTX-2B v0.9.5 uses guidance-free sampling. Higher CFG degrades coherence. |
| **Native VAE only** | SDXL VAE introduces color shift. Use `ltx_video_vae.safetensors`. |
| **49 frames is the 8 GB ceiling** | Safe; above 50 you hit OOM on most 8 GB setups. |
| **Junctions are the easy part** | Frame-locking gives SSIM 0.96+ boundaries out of the box. Intra-segment stability is the hard problem. |
| **Strength 0.40–0.45** | Above this the subject distorts; below it nothing moves. |
