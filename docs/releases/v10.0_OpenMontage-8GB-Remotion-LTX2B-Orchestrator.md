# OpenMontage on 8 GB: The Orchestrator That Finally Made My Pipeline a Product

> Date: 2026-07-19 · GPU: RTX 5060 Laptop 8GB · CUDA: cu132 · Framework: OpenMontage + Remotion + LTX-2B · OS: Windows 11 native
> Release: https://github.com/Ultramanmao/ai-pipeline-local/releases/tag/v10.0

## TL;DR

By mid-July I had nine image/video pipelines running on my 8 GB RTX 5060 Laptop. Every one worked. None of them produced a *product* — a real video you could post.

The gap wasn't generation. It was assembly. An animation PNG, a TTS MP3, a caption file — turning any of that into a postable video was manual, ad-hoc, and didn't scale.

**OpenMontage** (`calesthio/OpenMontage`, AGPLv3) is an open-source, agent-programmable video production system that chains "text-to-image → text-to-video → TTS → subtitles → editing → rendering" into a structured pipeline with checkpoints and approval gates.

The verdict: **not worth it for its built-in video model, but worth it for its assembly layer.** The bundled workflow ships with WAN 2.2 14B fp8 (~28 GB) — dead on 8 GB. But the Remotion + subtitles + music composition layer is genuinely useful, and it doesn't touch the GPU at all.

Three-phase result:

| Phase | Outcome | What it proved |
|-------|---------|----------------|
| P1 — setup + static demo | ✅ 23s Remotion video, 5.4 MB, ~90s render | Node/npm path works on Windows, no Chrome download needed |
| P2 — LTX-2B integration | ✅ End-to-end pipeline → 21s mp4, 4.2 MB | ComfyUI as material source, Remotion as renderer |
| P3 — (planned) | ⏳ | Real LTX-2B multi-segment relay into a full video |

**Bottom line: zero API keys, fully local, end-to-end video output in under two minutes render time.**

## The System, Stripped Down

OpenMontage has three layers:

| Layer | Content |
|-------|---------|
| Pipeline definitions | 13 YAML pipelines (cinematic, documentary-montage, clip-factory, talking-head, etc.) |
| Tool layer | 40+ tools: `comfyui_image`, `comfyui_video`, `kling`, `fal`, Remotion, Piper TTS, subtitles, stock data |
| Backlot | Local FastAPI web dashboard (uvicorn) — live progress + approval gates |

For me, only layer 2's Remotion and layer 1's structured pipeline concept mattered. The video-model tool itself is unusable on 8 GB, so I plugged in LTX-2B instead.

## Phase 1: `make setup`, or don't

The docs say `make setup`. On Windows MSYS2, `make` doesn't exist.

Ran the steps manually. Dependency install went clean:

```
npm install                   → 200 packages, registry.npmjs.org (no proxy needed)
Python .venv + requirements   → 48 packages
compositor-win32-x64-msvc     → remotion.exe + ffmpeg + 12 DLLs
```

First run of `python render_demo.py world-in-numbers` produced a clean 23-second, 690-frame h264 video at 5.4 MB. Pure Remotion — title cards, stat cards, bar charts, animated world map. No GPU. No API keys.

### Pitfall 1: Chrome Headless Shell download stalls

Google Storage times out from here (both direct and via proxy). The fix is to pass `--browser-executable msedge.exe` — MS Edge 150.0.4078.65 is already installed on Windows 11 and works as a drop-in replacement for the bundled Chromium.

### Pitfall 2: `npx remotion` not found

Changed the render script to call `node_modules/@remotion/cli/remotion-cli.js` directly instead of relying on npx resolution.

### Pitfall 3: npm install deleted 40 packages

Installing `@remotion/compositor-win32-x64-msvc` in isolation mutated `package.json` and dropped dependencies. `git checkout -- package.json` then re-install recovered everything.

## Phase 2: Pipe LTX-2B into Remotion

This is where it got real. The bundled `comfyui_video` workflow points at WAN 2.2 14B. I swapped it for my existing LTX-2B SOP (STG block 29 + StatNorm factor 1.0, strength 0.30, 768×512, 30 steps) and wired the output into Remotion.

### Deliverables

| File | Purpose |
|------|---------|
| `workflows/ltx2b_sop.json` | ComfyUI visual workflow (12 nodes / 17 connections) |
| `workflows/ltx2b_prompt_template.json` | API submission template |

| `demo-props/om-phase2.json` | Remotion short-video props |
| `projects/demos/renders/om-phase2.mp4` | Final product — 630 frames / 30 fps / 21s / 4.2 MB |

## Pitfall 4: ComfyUI API format silently broke

This was the one that stopped me cold. My LTX-2B relay scripts all returned HTTP 500. After tracing `execution.py`, the root cause:

Newer ComfyUI changed the `/prompt` endpoint contract:

**Old format (broken):**
```json
{"prompt": {node_id: {...}}}
```

**New format (required):**
```json
{"prompt": {"nodes": {node_id: {...}}, "client_id": "xxx"}}
```

The server calls `prompt.get("nodes")`, which returns `None` on the old format → `TypeError` → HTTP 500. Every legacy relay script, every test harness — all broken silently.

**If you have ComfyUI automation scripts, audit them now.** They may have been failing for weeks without anyone noticing.

---

## Pitfall 5: PIL can't write MP4

LTX-2B outputs an animated PNG. My first instinct: `Image.save(codec="libx264")`. This threw `KeyError: 'MP4'` because PIL has no libx264 encoder.

The fix is embarrassingly simple — defer to ffmpeg:

```bash
ffmpeg -i input.png -c:v libx264 -pix_fmt yuv420p output.mp4
```

Don't fight PIL on video encoding.

## Lessons

| Lesson | Detail |
|--------|--------|
| **OpenMontage is a factory, not a model** | The built-in WAN 14B won't fit on your GPU, but the Remotion assembly layer is the reason you bought the GPU |
| **Audit ComfyUI API scripts** | The `/prompt` format change is silent — old scripts return 500 without error messages |
| **ffmpeg > PIL for video** | PIL has no libx264 encoder. Never try `Image.save(codec="libx264")` |
| **P1 proves the path, not the product** | A 23-second pure-Remotion demo is the right first step — validate the renderer before wiring in generation |
| **MS Edge replaces Chrome** | On Windows 11, `--browser-executable msedge.exe` avoids the Chrome download stall entirely |

---

## The Real Story

OpenMontage ships with a 28 GB video model on its pipeline. On an 8 GB card, that model is unreachable. But nobody builds an orchestrator just for the model it ships with — you build it for the assembly it enables.

Plugging LTX-2B into Remotion turned a ComfyUI output directory full of isolated animation PNGs into something that could become a Bilibili video. That's the delta. The video-model choice is yours; the factory belongs to whoever can wire their generation into it.
